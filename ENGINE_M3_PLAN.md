## Sequencing (ratified 2026-06-22)

- **M3 = `reconcile` (primary) + `adjudicate` (secondary, Zingarelli-only).**
- **`typeset` split into its own milestone** (task #9, "M3b") — it is the heaviest
  deterministic step (1607 + `typography.py` 219 ≈ 1826 lines) and the *only* one that
  writes the protected live tree (`docs/index.html`, `docs/scan.html`, CSS —
  `typeset.py:1588-1602`). It earns a focused milestone with deploy-sync neutralization
  and the isolation test front-and-center.
- **Rationale:** `reconcile` is the deterministic *upstream* foundation. M2's `validate`
  already consumes `reconciled_chapters.json` (its word-count-preservation check), so we
  shipped a consumer before its producer; porting `reconcile` closes that inversion and
  lets us prove the `reconcile → validate` contract on engine-produced output.

## Discipline (applied — canonical: `engine/docs/port_discipline.md`)

The port discipline (three validation tiers + the isolation invariant, the
golden-as-discriminator change gate, one-way/two-way doors, the two registers) is canonical in
`engine/docs/port_discipline.md`. This section only *applies* it to M3.

- **Required tiers.** *reconcile* (deterministic) requires **equivalence + property +
  separability + isolation** — all four, not equivalence alone; it leads with equivalence.
  *adjudicate* requires **property/unit + isolation**; its equivalence golden is **deferred**
  (F2/D3), so property tests carry the floor. Green equivalence ≠ correct.
- **Change gate.** Within reconcile, a change that keeps the golden green is structure (free); a
  change that would move the golden is a regression (revert) or a logged divergence
  (`engine/docs/decisions/divergence_ledger.md`). Algorithm behavior is held to equivalence
  unless scan / period-dictionary ground truth licenses a specific change (logged).
- **M3 one-way doors** (decided on the record in the branch register when porting): reconcile's
  `reconciled_chapters.json` / `flagged_segments.json` step contracts, and the new
  `split_raw_chapters` plugin surface.
- **Branch register** (`engine/docs/decisions/branch_register.md`): **BR-001** (adjudicate↔oracle,
  revisit M6) is M3's binding deferral; consult the register before porting for any entry M3
  unblocks.

## Findings from pre-port verification (these reshape the validation strategy)

**F1 — `data/reconciled_chapters.json` is not a pristine `reconcile()` output.**
It was hand-patched out-of-band: `c66cfe3` ("clean known OCR/pipeline errors") changed
2 lines; `eb651a8` also touched it without rewriting its co-outputs (`flagged_segments.json`
last written by `d582ffc`; `reconciled_raw.txt` Apr 3). A clean reconcile run rewrites all
three together, so the live file has drifted from reproducible code output.
- M2's frozen validate input (`books/per_la_liberta/inputs/reconciled_chapters.json`) is
  byte-identical to the *hand-edited* live file (SHA `c0b49b8…`) — i.e. pinned to the
  post-edit version, 2 lines off the pristine output (`pre-c66cfe3` SHA `2cadae96…`).
- **Consequence:** the reconcile golden target is **generated by running live `reconcile.py`
  over frozen `copy{1,2,3}` in a temp dir** (the exact M2 fixture discipline), asserting
  `engine == that`. We do **not** diff against the hand-edited `data/` artifact. The drift
  vs. live `data/` is computed and logged by the fixture generator so it is documented,
  not hidden. (See D1.)

**F2 — `adjudicate`'s input is unrecoverable in paired form.**
`data/review_flags.json` is absent from the working tree *and* from `a0dd7d9` (the commit
that produced the committed `data/adjudication_results.json`). It survives only in *later*
commits (`f4c8cf7` … `922ed40`) — a different era that will not reproduce the committed
output. So adjudicate's golden cannot byte-reproduce `adjudication_results.json` from a
matching input. (See D3.)

**F3 — Substrate gaps for the port (grounded):**
- `rejoin_lines` (utils.py:298) — language-neutral line-rejoin (hyphen-aware). Not in
  engine → port to `util/text.py`.
- `split_into_chapters` (utils.py:230) — segments **raw OCR** into
  `[{id:p1_chNN, title, part, text}]`. The plugin's `chapter_identities()`
  (`lang/base.py:92`) parses *cleaned markdown* (`##`/`###`) — a different operation.
  reconcile needs raw-OCR segmentation, built on already-ported primitives
  (`is_chapter_heading` `italian.py:197`, `parse_chapter_number`) plus the Italian
  structural markers (PREFAZIONE / PARTE SECONDA / FINE DELLA PRIMA PARTE / PER LA LIBERTA,
  utils.py:249-266). → new Italian-owned plugin method (`split_raw_chapters`).
- `strip_boilerplate_copy1` / `copy2` are **vestigial aliases** of `strip_boilerplate`
  (utils.py:160-161) — identical; engine calls `lang.strip_boilerplate` at all three
  reconcile sites (reconcile.py:610,611,625).
- `collapse_spaces`, `normalize_for_comparison`, `atomic_write_json` — already in engine.

## reconcile port

### Language-neutral mechanics → `steps/reconcile.py` (port ~verbatim)
`score_word`*, `split_paragraphs`, `_is_near_duplicate` (rapidfuzz), `_split_merged_chapters`,
`align_paragraphs`, `_build_norm_map`, `_find_copy3_region`, `_find_copy3_text`,
`reconcile_words`, `reconcile_words_3way`, `_strip_page_markers`, `_find_page_for_paragraph`.
All SequenceMatcher / `statistics.median` / `rapidfuzz.fuzz` — deterministic, no LLM/network.
*(`score_word` carries one language seam — see D2.)*

### Config bindings (the parameterization map for reconcile)
| Live constant (file:line) | → destination |
|---|---|
| `PAGE_MARKER_RE = ⟨PAGE:N⟩` (reconcile.py:21) | **code default** — internal OCR tag protocol (plan §"internal tag protocols") |
| `score_word` accent set `àèìòùéÀÈÌÒÙ` (reconcile.py:35,40,52) | **LanguageProfile** — see D2 |
| `score_word` `"ii"→u` penalty (reconcile.py:48) | **code default**; Italian/Bodoni-leaning heuristic, candidate for SourceNoiseProfile later (noted, not elevated now) |
| page-map low-content skip `< 500` chars (reconcile.py:823) | **code default** (scan heuristic) |
| flash-before-pro page-map preference (reconcile.py:798-800) | **code default** |
| near-dup thresholds 75 / 90 / 200 (reconcile.py:66,85,90,97) | **code defaults** (tuning) |
| raw-OCR chapter segmentation + structural markers | **`lang/italian.py`** (F3) |

### Outputs (→ `BookWorkspace.data`)
`reconciled_chapters.json`, `flagged_segments.json`, `chapter_pages.json` (golden-asserted);
`reconciled_raw.txt` (emitted — golden? see D4).

`run(*, workspace, cfg, lang) -> dict` mirrors validate's M2 shape: reads `copy{1,2,3}` from
`ws.data`, writes outputs to `ws.data`. (Live `reconcile()` also supports a `--chapter`
subset/resume mode; engine `run` ports the full-book path; subset is a later flag, not M3.)

### Validation — equivalence tier (`test_reconcile_golden`, marker `golden`)
Reconcile's *equivalence* tier; its contract/property and separability tiers are in the Test
plan below.
- Freeze under `books/per_la_liberta/inputs/`: `copy1_raw.txt`, `copy2_raw.txt`,
  `copy3_raw.txt`, `copy3_flash_page_map.json` (+ `copy3_pro_page_map.json`).
- `_generate_reconcile_fixture.py` runs **live `reconcile.py`** over the frozen copies in a
  temp dir → `*_expected.json`; logs the drift vs. live `data/reconciled_chapters.json` (F1).
- Assert engine `reconcile.run()` == expected (3 JSON files): full equality + per-chapter
  spot checks (count, ids, a known flagged disagreement).
- **Determinism risk:** pin `rapidfuzz` (and its Levenshtein backend) in `engine/pyproject.toml`;
  a minor bump can shift `fuzz.ratio` and perturb near-dup decisions. Documented in the test.

F2 is the *signal*, not an obstacle: adjudicate's classification logic is a specification
(branches: noise / ner / compound / corrected / unknown) better pinned by branch unit tests
than by reproducing one committed output from a lost input. Port the code behind a thin
interface so M6 can later re-express it on the ≥2-of-N oracle — that fork goes in the **branch
register**; the door is closed deliberately at M6, not now.

## Open decisions (please audit inline)

**D1 (confirm).** reconcile golden target generated from live code over frozen inputs,
asserting `engine == that`; **not** diffed against the hand-edited `data/` file (same
discipline you ratified for M2's validate fixture). The drift vs. live `data/` is logged for

**D2.** `score_word`'s Italian accent set. Recommend a new `LanguageProfile.word_score_accents`
field seeded with the **exact** live set `àèìòùéÀÈÌÒÙ` (note: that set lacks `É`, which

**D3.** adjudicate golden, given F2. Options:
- (a) Recover a `review_flags.json` from a later commit (e.g. `f4c8cf7`), freeze it, generate
  a fresh `adjudication_results_expected.json` via live `adjudicate.py` → a *port-fidelity*

- (b) Port adjudicate's code in M3 with **unit tests only**; defer its golden to M4b, when
  cleanup legitimately regenerates `review_flags.json`.

Recommend **(b)**, with the adjudicate-vs-oracle fork logged in the **branch register** and the

## Test plan (three tiers + the isolation invariant)

- **Equivalence:** `test_reconcile_golden` (full-book, frozen copies, freshly generated
  reference). adjudicate's equivalence golden is deferred (F2/D3).
- **Contract / property (fast, no spaCy):** `score_word` ordering; `align_paragraphs` opcodes;
  `reconcile_words` 2-way + `reconcile_words_3way` majority/flag branches; `_is_near_duplicate`;
  `_strip_page_markers`; reconcile output satisfies `validate`'s input contract (the
  inversion-closing test — shape, not byte-equality with the hand-edited fixture); adjudicate
  classification branches (noise / ner / compound / corrected / unknown) on a synthetic flags
  fixture.
- **Separability:** raw-OCR chapter segmentation driven by config/plugin structural markers,
  not PLL-baked; tiny `copy{1,2,3}_raw.txt` under `books/synthetic/inputs/` so reconcile runs
  end-to-end on a non-PLL book (mirrors M2's synthetic proof). *Limit:* the synthetic is still
  Italian — structural injection only, not language generalization (deferred, BR-002).
- **Isolation (cross-cutting invariant, not a tier):** `reconcile.run` against a temp workspace;
  protected roots (`data/output/state/docs/static`) unchanged before/after (fast tier — reconcile
  needs no spaCy, so it runs without the integration marker).
